# Schedule Appointment Sample

Use this sample when a user wants a small appointment booking demo with both an admin side and a public side.

## Scope

The sample is intentionally independent from external calendars, email delivery, payment, LINE, or production app data.

It includes:

- Admin weekly appointment slot management as an Original Screen.
- Appointment slots scoped to the logged-in `user`.
- Admin public appointment URL button and dialog.
- Public appointment calendar that displays the target user's name at the top.
- Encrypted `user.id` in the public URL.
- Public booking form that creates a booking in an empty calendar cell.
- Dynamic seed slots for active users.

It excludes:

- Google Calendar or Outlook sync.
- Email notifications.
- Payment.
- Public cancellation.
- Multi-staff selection on the public side.
- Customer accounts or login.

## Fixed Names

- Slot table: `schedule_appointment_slots`
- Admin class: `schedule_appointment_slots_original_management`
- Public class: `schedule_appointment_public`
- Installer: `scripts/install_schedule_appointment.php`
- Manifest: `assets/schedule-appointment/schedule-appointment.json`
- Slot status constant array: `schedule_appointment_status`

The public class is separate from `public_pages` so the sample can be installed into projects that already have a `public_pages` class. In a production app, moving the entry functions into `public_pages` is acceptable.

## Admin Flow

`schedule_appointment_slots_original_management::run` shows a weekly calendar for the logged-in user.

The toolbar includes:

- Week navigation.
- A date jump control.
- `Public URL`, which opens a dialog with a URL generated by `$ctl->get_APP_URL("schedule_appointment_public", "calendar", ["user" => $ctl->encrypt($userId)])`.
- `Add Slot`, which opens the slot add dialog.

Each hour cell includes an add icon so the admin can create a slot at that time.

Each slot card includes:

- Time range.
- Status.
- Customer name and message when booked.
- Edit.
- Delete.

The admin side always writes `user_id` from the current login session, so another user's slots are not shown or edited from this screen.

## Public Flow

`schedule_appointment_public::calendar` accepts the encrypted `user` parameter, decrypts it, stores the user context in session, and shows the public appointment calendar.

The public calendar:

- Shows the target user's display name at the top.
- Shows weekdays from Monday to Friday.
- Shows 30-minute rows from 10:00 to 17:00.
- Displays `Select` only for future empty 30-minute cells.
- Displays existing non-cancelled schedule rows as busy time.

Visitors select an empty time, open `schedule_appointment_public::book`, enter name/email/phone/message, and submit to `schedule_appointment_public::save`.

On save, a new `schedule_appointment_slots` row is inserted:

- `status = booked`
- `starts_at` is the selected empty time
- `duration_minutes = 30`
- `customer_name`
- `customer_email`
- `customer_phone`
- `customer_message`
- `booked_at`

## Installer

From an FBP project root:

```bash
php fbp/docs/.agents/skills/fbp-app-samples/scripts/install_schedule_appointment.php
```

Options:

- `--root=/path/to/project`: run outside the project root.
- `--keep-data`: keep existing `schedule_appointment_slots` data.
- `--no-copy`: update DB/constant data without copying code.

The default installer copies code, upserts constants/tables/fields, resets sample schedule data, and inserts future booked/blocked seed rows for active users.

## Verification

After installation, run:

```bash
php fbp/cli.php app_call --json='{"class":"schedule_appointment_slots_original_management","function":"run"}'
php fbp/cli.php app_check --json='{"class":"schedule_appointment_slots_original_management","function":"run","checks":[{"path":"response_json.work_area.html","contains":"Public URL","label":"public url button"},{"path":"response_json.work_area.html","contains":"Schedule Appointment Slots","label":"screen title"}]}'
php fbp/cli.php app_call --json='{"class":"schedule_appointment_slots_original_management","function":"public_url_dialog"}'
php fbp/cli.php data_list --json='{"table":"schedule_appointment_slots","max":20}'
```

To test a public booking end to end, open the generated public URL, select an empty future cell, submit the booking form, and confirm a new `booked` row appears in `data_list`.
